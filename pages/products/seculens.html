What SecuLens Does

Complete Visibility Into Your Open-Source Supply Chain

SecuLens provides six layers of analysis to give you total control over your open-source risk posture.

Vulnerability Detection

Scans every direct and transitive dependency against our comprehensive vulnerability intelligence database of 250,000+ CVEs. Identifies known vulnerabilities with severity ratings, exploitability scores, and fix availability. Correlates findings with CISA KEV catalog for actively exploited vulnerabilities.

License Compliance

Detects license types for every dependency in your project, including dual-licensed components. Identifies license conflicts, copyleft contamination risks, commercial use restrictions, and attribution requirements. Generates compliance reports mapped to your organization's license policy.

SBOM Generation

Produces standards-compliant Software Bill of Materials in CycloneDX and SPDX formats. Includes complete dependency metadata, vulnerability associations, license information, supplier details, and cryptographic hashes for every component in your software supply chain.

Forensic License Analysis

Goes beyond manifest file declarations by scanning actual source code files for license headers, copyright notices, and license text patterns. Collects evidence from source files, README documents, and embedded notices. Assigns confidence scores to each license detection for audit-ready reporting.

Transitive Dependency Mapping

Resolves the complete dependency tree including all transitive (indirect) dependencies. Visualizes dependency chains to show how vulnerable components enter your project. Identifies dependency conflicts, version mismatches, and abandoned or unmaintained libraries.

Fix Guidance

Provides actionable remediation guidance for every vulnerability, including the minimum safe version, upgrade path analysis, and breaking change warnings. Identifies if a fix is available, if the component is end-of-life, or if an alternative library should be considered.

Forensic License Detection

What makes SecuLens unique: we do not just read license declarations from manifest files. We scan the actual source code to find embedded license headers, copyright notices, and license texts that may contradict or supplement declared licenses.

1

Source File Scanning

Every source file in every dependency is scanned for license headers, copyright notices, SPDX identifiers, and license text patterns. Covers files that manifest declarations miss entirely.

2

Evidence Collection

For each detected license, SecuLens records the exact file path, line number, matched text, and license pattern that triggered the detection. This creates an audit trail for legal review.

3

Confidence Scoring

Each license detection is assigned a confidence score (0-100%) based on match quality, source reliability, and corroboration from multiple files. High-confidence findings can be auto-approved; low-confidence findings are flagged for manual review.

4

Conflict Detection

Identifies cases where the declared license in a manifest file differs from what is found in the actual source code. Detects dual-licensing, license changes between versions, and copyleft contamination from transitive dependencies.

Interactive HTML Reports

Generate beautiful, self-contained HTML reports that can be shared with stakeholders, attached to compliance submissions, or archived for audit purposes. Reports are fully interactive with filtering, sorting, and drill-down capabilities.

Why SecuLens

Why Choose SecuLens Over the Competition

SecuLens vs Mend (WhiteSource)

While Mend offers broader ecosystem coverage, SecuLens provides deeper forensic license analysis with source-level evidence collection and confidence scoring. SecuLens operates fully on-premise with air-gapped support, while Mend requires cloud connectivity. For organizations under Indian regulatory frameworks, SecuLens provides built-in SEBI, RBI, and CERT-In compliance reporting that Mend does not offer.

SecuLens vs Black Duck

Black Duck is a strong enterprise SCA tool, but it comes with significant licensing costs and complexity. SecuLens offers comparable vulnerability detection and license analysis at a fraction of the cost, with simpler deployment and management. SecuLens generates self-contained interactive HTML reports, while Black Duck relies on a centralized dashboard that may not be accessible to all stakeholders.

SecuLens vs Snyk

Snyk excels in developer experience with its CLI and IDE integrations, but its on-premise deployment options are limited. SecuLens is built from the ground up for on-premise operation, ensuring your dependency data never leaves your network. SecuLens also provides forensic license detection at the source code level, while Snyk relies primarily on manifest file declarations.

SecuLens vs Sonatype

Sonatype offers strong integration with Maven/Gradle workflows through Nexus, but SecuLens covers more ecosystems with consistent analysis depth. SecuLens provides seven export formats including both CycloneDX and SPDX SBOMs, while Sonatype focuses primarily on its own reporting format. SecuLens confidence scoring for license detection gives legal teams audit-ready evidence that Sonatype does not provide.
