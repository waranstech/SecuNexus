General

SecuNexa is an enterprise-grade application security platform developed by Meteonic Innovation Pvt Ltd. The platform, called SecuNexus, combines four specialized security engines — SecuTrace (SAST), SecuLens (SCA), SecuStrike (DAST), and SecuShield (RASP) — into a single, unified solution. What makes SecuNexa unique is that it was built from the ground up to operate in on-premises and air-gapped environments, with no internet connectivity required for any of its core functions.

SecuNexa is developed by Meteonic Innovation Pvt Ltd, an Indian cybersecurity company headquartered in Bengaluru, Karnataka. The company specializes in building application security solutions for enterprises that require complete data sovereignty and the ability to operate in disconnected or air-gapped environments.

SecuNexa serves any organization that develops or maintains software applications, with particular strength in regulated and security-sensitive industries. Key sectors include banking and financial services (regulated by RBI and SEBI), defense and aerospace, government and public sector, critical infrastructure (energy, utilities, telecommunications), healthcare and pharmaceuticals, insurance, and technology companies with strict data governance requirements.

SecuNexa differentiates itself in several key ways: (1) It was built air-gapped from day one, not retrofitted for offline use. (2) It combines four security engines (SAST, SCA, DAST, RASP) into a single unified platform with a shared database and dashboard. (3) It includes an offline vulnerability database that provides the same detection coverage as cloud-connected tools. (4) It offers built-in compliance templates for Indian regulators (SEBI, RBI, CERT-In). (5) It is developed and supported locally in India, providing data sovereignty and local-timezone support.

Products

SecuTrace is our Static Application Security Testing (SAST) engine. It analyzes your source code without executing it to identify security vulnerabilities such as SQL injection, cross-site scripting (XSS), buffer overflows, hardcoded secrets, insecure cryptography, and more. It provides code-level remediation guidance, showing developers exactly where the vulnerability exists and how to fix it. SecuTrace integrates into your CI/CD pipeline to catch vulnerabilities early in the development lifecycle.

SecuLens is our Software Composition Analysis (SCA) engine. It scans your project's dependencies and third-party libraries to identify known vulnerabilities (CVEs), license compliance risks, and outdated packages. It uses an offline vulnerability database (mirroring NVD and other sources) that is updated via secure USB transfers. SecuLens generates a complete Software Bill of Materials (SBOM) and provides upgrade paths and alternative package recommendations for vulnerable dependencies.

SecuStrike is our Dynamic Application Security Testing (DAST) engine. It tests running applications by simulating real-world attacks from an external perspective. It identifies vulnerabilities like authentication flaws, server misconfigurations, injection vulnerabilities accessible via HTTP, and other runtime issues that cannot be detected by static analysis alone. SecuStrike is language-agnostic since it tests via HTTP and can scan any web application or API.

SecuShield is our Runtime Application Self-Protection (RASP) engine. It is deployed as an agent within your application's runtime environment and monitors application behavior in real-time. It can detect and block zero-day attacks, SQL injection attempts, XSS attacks, and other exploits as they happen — without requiring any code changes to your application. SecuShield currently supports Java, .NET, Node.js, and Python runtimes.

SecuTrace (SAST) supports Java, Python, JavaScript/TypeScript, C, C++, Go, C#/.NET, PHP, Ruby, Kotlin, Swift, and Scala. SecuLens (SCA) supports all major package managers including npm, Maven, Gradle, pip, NuGet, Go modules, Composer, Bundler, Cargo, and CocoaPods. SecuStrike (DAST) is language-agnostic since it tests via HTTP. SecuShield (RASP) supports Java, .NET, Node.js, and Python runtimes. We are continuously adding support for additional languages.

SecuNexa maintains an offline vulnerability database that mirrors multiple authoritative sources including the National Vulnerability Database (NVD), CVE records, GitHub Security Advisories, and package-specific advisory databases. This offline database is updated monthly (or more frequently for critical vulnerabilities) via encrypted USB update packages. The database also supports custom vulnerability definitions, allowing organizations to add internal or industry-specific vulnerability patterns.

Deployment

Yes. SecuNexus was designed from its first line of code to operate without any internet connectivity. All four engines, the vulnerability database, the dashboard, and all reporting capabilities work entirely offline. Installation is performed via encrypted media, and updates are delivered through signed USB packages. No data ever needs to leave your network — not for scanning, not for analysis, not for reporting, and not for vulnerability lookups.

SecuNexus offers three deployment models: (1) Full Air-Gap — zero external connectivity, USB-based updates, ideal for defense and government. (2) On-Premises with Controlled Updates — runs on your infrastructure with optional scheduled connectivity windows for updates via a proxy. (3) Private Cloud — deploys in your private cloud (AWS VPC, Azure VNet, GCP VPC) with container-based architecture and auto-scaling support. All three models keep your data on your infrastructure.

Minimum requirements: 8-core CPU, 32 GB RAM, 500 GB SSD storage, and a supported Linux distribution (RHEL 8+, Ubuntu 20.04+, or CentOS 8+). Recommended for production: 16-core CPU, 64 GB RAM, 1 TB SSD. PostgreSQL 14+ is used for the database (bundled with the installer). No internet connectivity is required. For large-scale deployments scanning hundreds of projects, we can provide custom sizing recommendations.

A standard on-premises deployment typically takes 3-5 business days from package delivery to first scan. Air-gapped deployments may take an additional week due to secure media handling and transfer procedures. The process includes: environment preparation (1 day), installation and configuration (1-2 days), integration setup (1-2 days), and training (1 day). Our deployment team handles all the setup work.

Compliance

SecuNexa includes built-in compliance templates for: SEBI Cybersecurity and Cyber Resilience Framework, RBI IT Governance and Information Security guidelines, CERT-In incident reporting requirements, OWASP Top 10, OWASP API Security Top 10, SANS Top 25, PCI DSS, GDPR (data protection aspects), ISO 27001 (Annex A controls), and NIST Cybersecurity Framework. Enterprise customers can also create custom compliance frameworks tailored to their internal policies or industry-specific regulations.

SecuNexa generates compliance reports by mapping your scan findings to specific controls and requirements within each compliance framework. Reports include: an executive summary with compliance status, detailed findings mapped to specific compliance controls, remediation recommendations prioritized by compliance impact, evidence documentation suitable for auditors, trend analysis showing compliance improvement over time, and exportable formats (PDF for management, CSV/JSON for integration, SARIF for developer tools). Reports are generated entirely locally and can be scheduled to run automatically after each scan.

Yes. SecuNexa maintains detailed vulnerability and incident data that can be used to generate pre-formatted reports aligned with CERT-In's reporting requirements. When a security incident is identified, you can quickly generate a report containing the affected systems, vulnerability details, potential impact assessment, and remediation steps — all in the format required by CERT-In. This significantly reduces the time needed to compile and submit incident reports within the 6-hour window.

There are three standard tiers — Starter, Professional, and Enterprise — with custom pricing available for unique requirements. There are no per-scan fees, no cloud hosting costs, and no surprise charges. Indian customers can be invoiced in INR.

Technical

SecuNexa integrates with CI/CD pipelines through multiple methods: (1) CLI — our command-line tool can be invoked as a build step in any CI/CD system. (2) Native plugins for Jenkins, GitLab CI, GitHub Actions, Azure DevOps, TeamCity, and Bamboo. (3) REST API for custom integrations. In a typical setup, scans are triggered automatically on each commit or pull request, and results are reported back to the pipeline. You can configure quality gates to fail builds if critical vulnerabilities are found. All CI/CD communication happens over your local network — no external connectivity needed.

SecuNexa supports multiple report formats: PDF (executive summaries and detailed technical reports), CSV (for spreadsheet analysis and data import), JSON (for programmatic consumption and integration), SARIF (Static Analysis Results Interchange Format, for IDE and DevOps tool integration), and XML. Enterprise customers can also create custom report templates with their organization's branding, specific sections, and tailored content for different audiences (executives, developers, auditors).

The offline vulnerability database is updated on a monthly release cycle. Each monthly update includes all new CVEs, updated severity scores, new advisory data, and refreshed package metadata from the previous month. For critical vulnerabilities (actively exploited, high-severity, wide impact), we provide emergency hotfix packages outside the regular cycle. Update packages are encrypted and digitally signed. For on-premises deployments with controlled connectivity, updates can be configured to download automatically during scheduled maintenance windows.

Yes. SecuNexa provides IDE plugins for Visual Studio Code and IntelliJ IDEA (including other JetBrains IDEs like WebStorm, PyCharm, etc.). The IDE plugins allow developers to run SAST scans directly from their editor, see inline vulnerability annotations, access remediation guidance without leaving the IDE, and view their personal fix queue. The plugins communicate with the local SecuNexus server over your internal network.

Yes. SecuNexa integrates with Jira, Azure Boards, GitHub Issues, and GitLab Issues. When vulnerabilities are discovered, they can be automatically created as tickets in your issue tracker with all relevant details — severity, affected file and line number, remediation guidance, and compliance mapping. As issues are resolved in SecuNexa, the linked tickets are updated automatically. For on-premises Jira and GitLab instances, all communication stays within your network.

Yes. SecuNexus provides a comprehensive REST API that allows you to trigger scans, retrieve findings, generate reports, manage projects and users, and integrate with any custom tooling. The API is documented with OpenAPI/Swagger specifications and includes authentication via API keys with RBAC enforcement. The API is served over HTTPS on your local network and is available in all deployment modes, including air-gapped.

SecuNexa employs multiple strategies to minimize false positives: (1) The Unified Risk Engine correlates findings across engines, which significantly reduces duplicate and false positive findings. (2) Findings can be marked as false positives, accepted risks, or suppressed with audit-trail documentation. (3) Custom rules allow you to exclude specific patterns that are known to be safe in your environment. (4) The platform learns from your suppression patterns to improve accuracy over time. Enterprise customers can also create organization-specific rule tuning profiles.
